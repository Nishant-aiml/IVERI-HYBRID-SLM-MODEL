### Bug 2: `sft_runner.py` — Invalid Format Strings
```python

# Line 395: logger.info("... top1=%.2%%", ...)
```

### Bug 3: SFT Loss Masking Not Working
```
SFTByteDataset returns (x, y) tuples without loss_mask.
sft_runner.py defaults to loss_mask = ones when absent.
Result: prompt tokens are never masked during SFT training.
This defeats the purpose of SFT (should only train on response tokens).
```
**Severity**: 🔴 HIGH — SFT training will be fundamentally wrong
