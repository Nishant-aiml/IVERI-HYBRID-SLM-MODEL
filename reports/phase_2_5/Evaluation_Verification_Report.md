## 1. Metric Calculations

Verification of language modeling evaluation metrics is completed under `test_perplexity_computation` and `test_perplexity_nan_handling`.

| Metric | Calculation | Verification Status |
|---|---|---|
| **Cross Entropy** | Stable cross entropy loss sum with padding option. | Verified |
| **Negative Log Likelihood** | Summed negative log-likelihood over active tokens. | Verified |
| **Perplexity** | Exponential of mean NLL loss with NaN/Inf guards. | Verified |

### Numerical Stability & Sanitization
The `PerplexityEvaluator` resolves potential overflow or division-by-zero errors:
- If no active tokens are processed, returns 0.0 metrics.
- Exponentiations for perplexity that would exceed floating-point precision trigger an `OverflowError` catch and return `0.0`.
- Values that are `NaN`, `+Inf`, or `-Inf` are sanitized to `0.0`.

### Decoding Telemetry
Every generation pass records the following performance markers:
- **Latency**: Total wall-clock time in seconds.
- **Throughput**: Generated bytes per second.
- **Sequence Length**: Average generated byte length.
- **Early Stopped**: Percentage of sequences hitting the `EOS_BYTE` (0x02) boundary prior to hitting `max_new_bytes`.
